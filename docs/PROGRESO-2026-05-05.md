### 2) Sincronizacion mensual alineada al negocio

Se ajusto syncMesActual para usar el mes a cobrar (mes anterior al actual), y se reforzo la inclusion de clientes en estados especiales:

- inactivo
- factura_enviada

Para factura_enviada se mantuvo visibilidad sobre el periodo realmente adeudado (targetMes basado en mes_por_pagar cuando aplica).

Archivo clave:

- [app/Http/Controllers/Inbox/FacturaEnvioController.php](app/Http/Controllers/Inbox/FacturaEnvioController.php)

### 3) Importador CSV de clientes mas robusto

Se extendio import:customers para interpretar mejor los archivos reales de operacion:

- Lectura de columnas explicitas de tracking de pago:
  - pago_estado
  - mes_pagado
  - mes_por_pagar
- Derivacion automatica desde columnas mensuales ENERO..DICIEMBRE cuando no hay columnas explicitas.
- Normalizacion de estados textuales:
  - pendiente
  - factura_enviada
  - pagado
  - inactivo
- Mapeo de CONTACTO a contact_name.
- Fallback de USUARIO como contact_email cuando viene correo en esa columna.
- Correccion del orden de inicializacion de monthMap para evitar derivaciones nulas.

Archivo clave:

- [routes/console.php](routes/console.php)

### 4) Inferencia de servidor cuando el CSV no trae columna SERVIDOR

Se detecto un caso real en ATLANTIS FAST donde el encabezado no incluia SERVIDOR. Se aplico fallback automatico:

1. Intentar inferir servidor desde link (dominio).
2. Si no se puede, usar nombre del archivo CSV como respaldo.

Esto evita guardar servidor null cuando el dataset trae evidencia indirecta suficiente.

Archivo clave:

- [routes/console.php](routes/console.php)

## Diagnostico y Resolucion de Incidencias

### Incidencia A: Clientes importados no visibles en pestana Clientes

Causa raiz:

- La pestana Clientes depende de pagos_mensuales (no solo de customers).

Acciones:

- Se verifico estructura de datos y se poblaron/validaron registros relacionados.
- Se confirmo conteo por mes para garantizar aparicion en bandeja.

### Incidencia B: Inactivos y enviados no aparecian en sincronizacion

Acciones:

- Se modifico filtro de sincronizacion para incluirlos explicitamente.
- Se ajusto targetMes en enviados para reflejar deuda real.

### Incidencia C: Filtro Servidor no visible tras cambio

Causa:

- Assets viejos en navegador/bundle no actualizado.

Accion:

- Compilacion frontend realizada con exito y verificacion del bundle generado.
